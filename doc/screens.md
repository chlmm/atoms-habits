# Atoms 界面规格

> 移动端优先设计，当前以 Linux 桌面运行，但布局风格对标 Android。
> 后续用 `LayoutBuilder` + `defaultTargetPlatform` 适配桌面端。

## 1. 引导页 (OnboardingPage)

**路由**: `/onboarding` (初始路由)
**文件**: `lib/pages/onboarding_page.dart`
**导航**: PageView 横滑，5 页 → 主页或创建目标页

```
┌──────────────────────────────────┐
│                                  │
│   一切从你想达成的结果开始。      │
│                                  │
│   ┌──────────────────────────┐   │
│   │     完成双力臂            │   │ ← 演示目标卡片
│   └──────────────────────────┘   │
│                                  │
│   比如可以是一个具体的成果：      │
│   · 减掉 10 公斤                 │
│   · 每天醒来精力充沛             │
│   · 读完 20 本书                 │
│                                  │
│  [跳过演示，我自己来]  [下一步→]  │
│                                  │
│         ● ○ ○ ○ ○               │ ← 页面指示器
└──────────────────────────────────┘
```

### 5 页内容

| 页码 | 标题 | 内容 |
|------|------|------|
| 1 | 一切从你想达成的结果开始 | 演示目标名 "完成双力臂" |
| 2 | 然后拆成阶段性关口 | 4 个里程碑列表，第一个自动激活 |
| 3 | 针对第一个关口，列出需要做的行为 | 6 个行动项 |
| 4 | 把这些行动按频率组织成习惯 | 2 个习惯卡片（练背计划/核心训练）+ 安全阀 |
| 5 | 学会了。现在该你了 | 两个选择卡片 |

### 第 5 页选择

```

│          🚀                      │
│    学会了。现在该你了。           │
│                                  │
│  ┌────────────────────────────┐  │
```

### 2.5 ☰ 抽屉 (Drawer) — MemoFlow 风格

```
┌──────────────────────────┐
│ Atoms                 ⚙  │  ← 应用名左 + 设置图标右上
│                          │
│  12     3      28       │  ← 大号统计数字
│ 习惯   目标    天        │
│                          │
▢▢▣▢ ▢▢▣▣ ▢▢▢▢ ▢▣▢▢ ... │  ← GitHub 风格热力图 (~18周×7天)
▣▢▢▢ ▢▣▣▢ ▢▢▣▢ ▢▢▣▢ ... │     色深=打卡强度, 4级色阶+空格
▢▣▢▣ ▣▢▢▢ ▢▢▢▣ ▢▢▢▢ ... │     绿色块可点击 → 弹出当天详情
...                        │
 9月   10月   11月  12月   │  ← 月份标签
│ ──────────────────────── │
│ ● 目标面                  │  ← 当前选中 = primary 色
│ ○ 习惯面                  │
│                          │
│            v0.1          │  ← 底部版本号
└──────────────────────────┘

交互:
- 点击 ⚙ → 关闭抽屉 → 打开设置页(待实现)
- 点击 目标面/习惯面 → 切换面 + 关闭抽屉
- 热力图绿色块(count>0) → 关闭抽屉 → 弹出 DayDetailSheet (见 § 弹窗)
- 热力图灰色空块(count=0) → 无响应
- 长按任意格子 → Tooltip 显示 "YYYY-MM-DD: N次打卡 / 无打卡"
```

**结构层次**:
| 区域 | 组件 | 文件 | 说明 |
|------|------|------|------|
| Header | Row(Atoms + ⚙ IconButton) | main_page.dart | 品牌名 + 设置入口 |
| Stats | DrawerStats | widgets/drawer_stats.dart | 3 列大号数字(习惯数/目标数/活跃天数) |
| Heatmap | DrawerHeatmap | widgets/drawer_heatmap.dart | GitHub 贡献图, ~126天, 4级色阶, 可点击 |
| Nav | ListTile ×2 | main_page.dart | 目标面 / 习惯面 切换 |
| Footer | Text(v0.1) | main_page.dart | 居中版本号 |

**数据源**:
- 统计数字: `HabitService.getActiveHabitCount()`, `GoalService.getAllGoals()`, `HabitService.getActiveDaysCount()`
- 热力图: `HabitService.getDailyCheckinCounts()` (SQL 聚合 logs+habits)
- 点击详情: `HabitService.getHabitsWithStatusForDate()` (LEFT JOIN habits+logs)

## 设计原则

1. **移动端优先**: ☰ 抽屉 + 底部大 FAB，无侧边栏
2. **内联切换**: 搜索、面切换都在同一页面内通过状态驱动显示/隐藏
3. **渐进引导**: PageView 步骤式向导，不可跳步
4. **安全阀优先**: 每个习惯都有两分钟退路
5. **永不连续错过**: 昨天跳过 + 今天未完成时橙色警告
6. **身份认同**: 习惯完成次数触发认知转变提示
