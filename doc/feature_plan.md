# 原子习惯 App 功能规划

> 基于 James Clear 《原子习惯》方法论的习惯系统搭建器。
> 核心定位：帮助用户**设计系统**而非仅追踪打卡。
> 桌面端做规划，移动端做执行。

---

## 一、核心理念

| 传统习惯 App | 原子习惯 App |
|-------------|-------------|
| 到点提醒 → 打卡 ✓ | 先定义身份 → 设计环境 → 降低门槛 → 自然发生 |
| 关注"做没做" | 关注"为什么会做 / 为什么不会做" |
| 连续天数焦虑 | 累计进步曲线 + "绝不连续错过两次" |
| App = 闹钟 + 清单 | App = 教练 + 系统设计师 |

---

## 二、四定律 → 功能模块

### 定律一：让它显而易见

| 功能 | 说明 |
|------|------|
| **身份声明** | 用户首先回答"我想成为怎样的人"，而非"我要做什么"。例如："我是一个每天写作的人" |
| **习惯叠加** | 引导格式："在 [现有习惯] 之后，我会 [新习惯]"。可视化叠加链，形成习惯树 |
| **环境设计** | 可视化画布，拖拽布置触发点。例如：把跑鞋放在门口、把手机移出卧室 |
| **具体化表单** | "我会在 [时间] 在 [地点] 做 [具体行为]"，降低认知负荷 |
| **习惯列表视图** | 按时间、地点、上下文分组展示当前所有习惯 |

**桌面端侧重：** 环境设计画布、习惯叠加链可视化  
**移动端侧重：** 时间地点提醒、快捷查看"此时此地该做什么"

---

### 定律二：让它有吸引力

| 功能 | 说明 |
|------|------|
| **喜好绑定** | 用户设定：做完习惯 → 解锁喜欢的事。例如："跑完步才能看一集剧" |
| **加入群体** | 可选：创建或加入小的习惯小组（3-5 人），共享进度但不制造比较焦虑 |
| **身份证据墙** | 视觉化展示"我已经是一个 ____ 的人"的证据和里程碑 |
| **仪式感** | 为重要习惯设置简短的前置仪式（深呼吸、泡杯茶），触发心流状态 |

**桌面端侧重：** 设定绑定规则、查看群体进度  
**移动端侧重：** 执行仪式、解锁奖励提示

---

### 定律三：让它简单易行

| 功能 | 说明 |
|------|------|
| **两分钟模式** | 每个习惯都有一个"两分钟版本"。日常追踪两分钟版本而非完整版本 |
| **环境阻力清单** | 提前准备检查表（器材、场地、材料），减少启动摩擦 |
| **专注启动** | 打卡界面强调"只需要开始"，而非"必须完成" |
| **一键降级** | 今天太累？一键切换到最低版本，保持连续性 |
| **专注模式** | 开始习惯时进入专注计时，屏蔽通知 |

**桌面端侧重：** 设计两分钟版本、准备检查清单  
**移动端侧重：** 一键启动、专注计时、降级模式

---

### 定律四：让它令人满意

| 功能 | 说明 |
|------|------|
| **累计进步曲线** | 不是连续天数，而是累计完成次数曲线。中断不被清零，只是斜率变化 |
| **绝不连续错过两次** | 核心规则：可以错过一次，但不允许连续两天都错过。此为唯一硬约束 |
| **习惯合约** | 写下承诺、设定如果失败就执行的后果，可分享给搭档 |
| **习惯回顾** | 每周回顾：哪些习惯在自动运行？哪些需要重新设计系统？ |
| **里程碑** | 不是 7/21/66 天固定里程碑，而是与用户个人进步绑定的自定义节点 |

**桌面端侧重：** 回顾仪表盘、合约设定、里程碑规划  
**移动端侧重：** 即时反馈动画、每日回顾卡片

## 三、双端角色划分

```
桌面端（Linux/Mac/Windows）          移动端（Android/iOS）
═══════════════════════            ═══════════════════
系统设计                            执行
  ├ 创建/编辑习惯                      ├ 查看今日习惯
  ├ 设计环境布局                       ├ 一键启动 + 专注计时
  ├ 习惯叠加链可视化                    ├ 完成反馈动画
  ├ 回顾仪表盘                         ├ 降级/跳过
  └ 合约与里程碑设定                    └ 提醒通知

数据层（Go 后端 — 本地 or 远程共享）
  ├ 习惯模型 CRUD
  ├ 统计分析引擎（趋势、预测、最优时段）
  ├ 数据同步（桌面 ↔ 移动）
  └ 备份导出
```

## 四、用户核心流程

```
首次使用
  │
  ├─ 1. 身份声明  "我想成为____的人"
  │
  ├─ 2. 系统设计
  │     ├ 习惯叠加：在[现有习惯]之后，我会[新习惯]
  │     ├ 两分钟版本：最小可行版本是什么？
  │     ├ 环境设计：我需要准备什么？
  │     └ 具体化：什么时间？什么地点？
  │
  └─ 3. 开始执行
        ├ 每日：两分钟入口 → 自动记录
        ├ 每周：回顾仪表盘 → 调整系统
        └ 长期：身份证据累积 → 自然成为

日常使用
  │
  ├─ 打开 App → 看到"此时此地该做什么"（环境感知）
  ├─ 一键进入专注模式
  ├─ 完成后即时反馈（累计曲线更新、喜好解锁）
  └─ 从不过度打扰（只有关键提醒 + 绝不连续错过两次告警）
```

## 五、功能优先级

| 优先级 | 功能 | 对应定律 |
|--------|------|----------|
| P0 | 习惯 CRUD + 两分钟模式 | 定律三 |
| P0 | 累计进步曲线 + 绝不连续错过两次 | 定律四 |
| P1 | 身份声明 + 习惯叠加 | 定律一 |
| P1 | 喜好绑定 | 定律二 |
| P1 | 每周回顾仪表盘 | 定律四 |
| P2 | 环境设计画布 | 定律一 |
| P2 | 专注计时模式 | 定律三 |
| P2 | 习惯小组 | 定律二 |
| P3 | 习惯合约 | 定律四 |
| P3 | 统计分析 + 趋势预测 | Go 后端能力 |
| P3 | 数据同步（桌面 ↔ 移动） | 远程部署 |
| P3 | 移动端提醒 + 快速执行 | Android 端 |
